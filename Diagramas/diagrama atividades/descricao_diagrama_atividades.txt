Diagrama Atividades
O diagrama de Entidades é composto por três atores que interagem com toda aplicação, sendo eles o cliente, o sistema e o vendedor.
Para inciar aplicação tanto o cliente como o vendedor tem como sua primeira atividade fazer o login, o sistema confirma a cessão iniciada e com um nó de decisão o sistema confirma a cessão inciada ou em caso de falha o mesmos voltam á mesma atividade login para confirmar os dados inseridos.
Confirmado o sucesso de login por parte do sistema, desperta então os devidos garfos para representar os fluxos do respetivo cliente e vendedor.

Fluxos Cliente
O cliente tem acesso a três atividades, sendo elas o Histórico, carrinho, e a lista de produtos.
Em caso de o cliente prentender executar atividade do Histórico, vai ter acesso á ação para listar todos os pedidos efetuados até á data atual, terminando o fluxo com ação apresentada.
Se a atividade do cliente for listar Produtos, segue-se ação de poder selecionar o produto que automáticamente é adicionado na atividade do carrinho.
Na atividade do carrinho o cliente pode  executar duas ações, efetuar um pedido ou cancelar um pedido. No caso de efetuar um pedido o mesmo será encaminhado  ao sistema que encaminhará para o vendedor. O fluxo terminará quando o sistema notificar que o pedido está na lista de pedidos do vendedor.
Se o cliente executar ação para cancelar um pedido, a ação para cancelar o pedido será executada e termina o fluxo.

Fluxos Vendedor
O vendedor tem acesso a três atividades, sendo elas o Histórico de Pedidos, os Produtos, e os pedidos
No caso da atividade ser o Histórico de Pedidos é executada a ação para listar todos os pedidos e o fluxo termina.
Se a atividade for Produtos, então o vendedor tem três ações, inserir produtos, alterar produtos, apagar produtos, as ações seram enviadas para o sistema e o mesmo se encarrega de executar ação pretendida pelo vendedor, e é dado como terminado o fluxo pelo Sistema.
No caso da ação pretendida for os pedidos, a ação preparação é executada, segue-se ação de entrega e concluido, onde é enviada uma ação para o sistema notificar o cliente, seguindo-se ação de cliente notificado, o pedido está pronto a ser entregue, o cliente está notificado, fluxo termina.
